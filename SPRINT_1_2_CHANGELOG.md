# SPRINT 1-2 CHANGELOG

Дата фиксации: 04 марта 2026

## Sprint 1: Базовый каркас и доменная модель

1. Реализована ролевая модель (`teacher`, `program_leader`, `dean`, `umu`, `admin`).
2. Реализованы сущности:
   `Course`, `Topic`, `TopicLiterature`, `TopicQuestion`, `Syllabus`, `SyllabusTopic`.
3. Реализованы страницы:
   курсы, темы, список силлабусов, карточка силлабуса.
4. Реализован базовый workflow статусов и контроль прав переходов.
5. Добавлены журналы изменений и статусов.

## Sprint 2: Ускорение, удобство, устойчивость

### 2.1 Конструктор и UX-поток
1. Включено редактирование тем силлабуса через конструктор.
2. Включено редактирование деталей силлабуса.
3. В карточке силлабуса добавлен явный блок шагов:
   `Редактировать темы` -> `Редактировать детали` -> `Отправить на проверку ИИ`.
4. Интерфейс уточнен: конструктор и импорт файла обозначены как два альтернативных сценария.

### 2.2 Качество и валидация
1. Перед отправкой в ИИ добавлена структурная валидация:
   без обязательной структуры документ не уходит в `ai_check`.
2. Добавлены проверки форматов загружаемых файлов (`.pdf`, `.doc`, `.docx`).
3. Добавлены и обновлены тесты на ключевые бизнес-сценарии.

### 2.3 ИИ-модуль
1. Добавлена быстрая rule-based проверка (`fast-rules`) для ускорения.
2. Добавлен детектор нерелевантных документов (not-syllabus guard).
3. Оптимизирован pipeline обработки и логика fallback.
4. Улучшена устойчивость worker-процесса и обработка ошибок.

## Итог Sprint 1-2

1. Проект покрывает полный цикл:
   `банк тем -> сборка/импорт -> ИИ -> декан -> УМУ -> утверждение`.
2. Снижен ручной труд преподавателя при повторной подготовке силлабусов.
3. Повышена прозрачность согласования и контроль версий.
4. Система готова к демонстрации дипломного сценария "как есть".
